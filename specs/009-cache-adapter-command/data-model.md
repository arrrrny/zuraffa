# Data Model: Cache Adapter Command

**Feature**: Cache Adapter Command
**Date**: 2026-06-12

## Overview

This document defines the logical components and data flows for the `zfa cache adapter` command. Since this is a developer tooling feature (not a data-centric application), the "data model" describes the system components involved in the adapter generation process.

## System Components

### CLI Command: `zfa cache adapter <EntityName>`

The sub-command is auto-generated by `CapabilityCommand` from the capability's `inputSchema`.

**Usage**:
```bash
zfa cache adapter <EntityName> [--build] [--dry-run] [--force] [--verbose]
```

**Example**:
```bash
zfa cache adapter Product --build
```

**Argument mapping**:
- `<EntityName>` (positional, required) â†’ maps to `name` in inputSchema
- `--build` â†’ maps to `build` in inputSchema (runs `zfa build` after registrar update)
- `--dry-run` â†’ maps to `dryRun` in inputSchema
- `--force` â†’ maps to `force` in inputSchema
- `--verbose` â†’ maps to `verbose` in inputSchema

### Entity Types and Their Detection

| Entity Type | Detection Method | Adapter Generation |
|-------------|-----------------|-------------------|
| Entity class (Zorphy-annotated) | Found in `domain/entities/{snake}/{snake}.dart` with `@Zorphy` annotation | `AdapterSpec<EntityName>()` |
| Enum class | Found in `domain/entities/{snake}/{snake}.dart` as an `enum` type | `AdapterSpec<EnumType>()` |
| Sub-entity | Discovered via field type scanning with `EntityAnalyzer` | `AdapterSpec<SubEntity>()` |
| Primitive type (String, int, double, bool, DateTime, etc.) | Known via `KnownTypes.isExcluded()` | Skipped â€” no adapter needed |
| External library type | Not in `domain/entities` | Skipped â€” assumed to have its own adapter |

### Hive Registrar File Structure

The `hive_registrar.dart` file is the central output. It follows this structure:

```dart
// Auto-generated Hive registrar
import 'package:hive_ce_flutter/hive_ce_flutter.dart';
// ... entity-specific imports ...
import '../domain/entities/product/product.dart';

part 'hive_registrar.g.dart';

@GenerateAdapters([
  AdapterSpec<Product>(),
  AdapterSpec<Category>(),
  // ... more adapter specs ...
])
extension HiveRegistrar on HiveInterface {
  void registerAdapters() {
    registerAdapter(ProductAdapter());
    registerAdapter(CategoryAdapter());
    // ... more registrations ...
  }
}

extension IsolatedHiveRegistrar on IsolatedHiveInterface {
  void registerAdapters() {
    registerAdapter(ProductAdapter());
    registerAdapter(CategoryAdapter());
    // ... more registrations ...
  }
}
```

### State Transitions

The capability does not maintain persistent state. Each invocation is stateless:

1. **Input**: Entity name string
2. **Process**: Analyze entity â†’ discover sub-entities â†’ regenerate registrar â†’ optionally build
3. **Output**: ExecutionResult with success/failure, list of registered entities, generated files

### Validation Rules

| Rule | Description | Enforcement |
|------|-------------|-------------|
| Entity exists | The specified entity must have a file at the expected path | Error message listing available entities if not found |
| No duplicates | Same entity cannot appear twice in the registrar | Deduplication by entity name during file generation |
| Circular references | Entity graph cannot have cycles | Track visited entities in a `Set<String>` during discovery |
| File permissions | Must be able to read/write files | User-friendly error with file path |
