# 🧬 Morphy Integration Guide

**Zuraffa + zikzak_morphy = Perfect Serialization**

> Why Morphy over Freezed? Because it's built for Primitive JSON Forever™

---

## 🎯 Philosophy

### Primitive JSON Forever

Zuraffa enforces **simple, flat JSON structures** using Morphy:

**✅ Allowed Types:**
- `String`
- `int`
- `double`
- `bool`
- `DateTime`
- `List<T>` (where T is primitive or entity)
- Nested entities (with `$` prefix)

**❌ Not Allowed:**
- `Map<K, V>` with non-String keys
- Complex nested structures
- Union types (use sealed classes instead)
- Optional fields as business logic (use proper types)

## 🏗️ Generated Code Structure

### Basic Entity

**Input JSON:**
```json
{
  "id": "prod-123",
  "name": "Wireless Headphones",
  "price": 99.99,
  "inStock": true
}
```

**Generated Morphy Entity:**
```dart
// Generated by Zuraffa on 2025-11-14T...
// DO NOT EDIT - Regenerate with: zuraffa create entity Product

import 'package:zikzak_morphy/zikzak_morphy.dart';

part 'product.morphy.dart';
part 'product.g.dart';

@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  String get name;
  double get price;
  bool get inStock;
}
```

**What gets generated:**
1. `product.dart` - Your abstract class
2. `product.morphy.dart` - Morphy's implementation
3. `product.g.dart` - JSON serialization

## 🔄 Morphy vs Freezed

| Feature | Morphy | Freezed |
|---------|--------|---------|
| **Syntax** | Clean `$` prefix | Verbose mixins |
| **JSON** | Built-in | Requires json_serializable |
| **Inheritance** | ✅ Full support | ❌ Limited |
| **copyWith** | ✅ Generated | ✅ Generated |
| **Equality** | ✅ Generated | ✅ Generated |
| **Nullability** | Explicit | Explicit |
| **Philosophy** | Primitive JSON | Union types |
| **ZikZak AI** | ✅ Optimized | ❌ Not optimized |

## 📝 Usage Examples

### 1. Simple Entity

```dart
@Morphy(generateJson: true)
abstract class $User {
  String get id;
  String get name;
  String get email;
  DateTime get createdAt;
}

// Use it:
final user = User.fromJson(json);
final updatedUser = user.copyWith(name: 'New Name');
print(user.toJson());
```

### 2. Entity with Lists

```dart
@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  String get name;
  List<String> get tags;        // List of primitives
  List<String> get imageUrls;   // List of primitives
}
```

### 3. Nested Entities

```dart
@Morphy(generateJson: true)
abstract class $Order {
  String get id;
  $Customer get customer;              // Single nested entity
  List<$OrderItem> get items;          // List of entities
  DateTime get createdAt;
}

@Morphy(generateJson: true)
abstract class $Customer {
  String get id;
  String get name;
}

@Morphy(generateJson: true)
abstract class $OrderItem {
  String get id;
  $Product get product;                // Nested entity
  int get quantity;
}
```

## 🔧 Advanced Patterns

### 1. Computed Properties

```dart
@Morphy(generateJson: true)
abstract class $Cart {
  List<$CartItem> get items;

  // Morphy-generated class can add computed properties
}

// In cart.morphy.dart (generated):
extension CartExtension on Cart {
  double get total => items.fold(0, (sum, item) => sum + item.total);
  int get itemCount => items.length;
}
```

### 2. Validation

```dart
@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  String get name;
  double get price;

  // Add validation in your UseCase layer, NOT in entity
}

// In your UseCase:
class CreateProductUseCase {
  Result<Product, ValidationFailure> execute(Map<String, dynamic> json) {
    // Validate before creating entity
    if (json['price'] <= 0) {
      return Failure(ValidationFailure('Price must be positive'));
    }

    final product = Product.fromJson(json);
    return Success(product);
  }
}
```

### 3. Immutability

Morphy entities are **immutable by default**:

```dart
final product = Product.fromJson(json);

// ❌ Can't do this - no setters!
// product.price = 79.99;

// ✅ Must use copyWith
final updated = product.copyWith(price: 79.99);
```

## 🚀 Zuraffa Best Practices

### 1. Always Have an `id` Field

```dart
@Morphy(generateJson: true)
abstract class $Product {
  String get id;  // ✅ Required! Zuraffa enforces this
  // ... other fields
}
```

**Why?**
- Cache management needs IDs
- Repository operations need IDs
- Testing needs IDs

### 2. Use DateTime for Timestamps

```dart
@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  DateTime get createdAt;     // ✅ Good
  DateTime get updatedAt;     // ✅ Good
  // NOT String createdAt;    // ❌ Bad
}
```

### 3. Keep Entities Flat

```dart
// ✅ Good - Flat structure
@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  String get name;
  double get price;
  String get categoryId;      // Reference by ID
}

// ❌ Bad - Deep nesting
@Morphy(generateJson: true)
abstract class $Product {
  String get id;
  $Category get category;
  // ^^ This couples Product to Category
}
```

**Instead, use UseCases to compose:**

```dart
class GetProductWithCategoryUseCase {
  Future<Result<ProductWithCategory, Failure>> execute(String id) async {
    final product = await productRepo.getById(id);
    final category = await categoryRepo.getById(product.categoryId);

    return Success(ProductWithCategory(
      product: product,
      category: category,
    ));
  }
}
```

### 4. Nullable Fields Only When Necessary

```dart
@Morphy(generateJson: true)
abstract class $Product {
  String get id;               // ✅ Required
  String get name;             // ✅ Required
  double? get discount;        // ✅ Nullable - optional discount
  String? get description;     // ✅ Nullable - optional field
}
```

## 🧪 Testing Morphy Entities

Zuraffa auto-generates tests for:

### 1. JSON Serialization

```dart
test('fromJson creates valid Product', () {
  final json = {'id': 'prod-1', 'name': 'Test', 'price': 99.99};
  final product = Product.fromJson(json);

  expect(product.id, 'prod-1');
  expect(product.name, 'Test');
  expect(product.price, 99.99);
});
```

### 2. Roundtrip Testing

```dart
test('fromJson -> toJson roundtrip preserves data', () {
  final originalJson = {'id': 'prod-1', 'name': 'Test', 'price': 99.99};
  final product = Product.fromJson(originalJson);
  final serializedJson = product.toJson();
  final deserializedProduct = Product.fromJson(serializedJson);

  expect(deserializedProduct.toJson(), equals(originalJson));
});
```

### 3. copyWith Testing

```dart
test('copyWith creates new instance with updated fields', () {
  final product = Product.fromJson(json);
  final updated = product.copyWith(price: 79.99);

  expect(updated.price, 79.99);
  expect(updated.id, product.id); // Other fields unchanged
  expect(product.price, 99.99);   // Original unchanged
});
```

## 🔍 Common Issues & Solutions

### Issue 1: Missing Part Files

**Error:**
```
Target of URI doesn't exist: 'product.morphy.dart'
```

**Solution:**
```bash
# Run build_runner to generate .morphy.dart files
dart run build_runner build --delete-conflicting-outputs
```

### Issue 2: Wrong $ Prefix

**Error:**
```dart
// ❌ Wrong
abstract class Product { ... }

// ✅ Correct
abstract class $Product { ... }
```

### Issue 3: Nested Entity Not Found

**Error:**
```
Undefined class '$Customer'
```

**Solution:**
```dart
// Add import at top of file
import 'customer.dart';

@Morphy(generateJson: true)
abstract class $Order {
  $Customer get customer;  // Now found!
}
```
